The data collection consists of a Product_Collection that lists the data products found in the data collection and a Product_Observational for each data product. PDS4 uses four fundamental data structures:

  1. Array (images, spectral cubes)
  2. Table (binary and/or character fixed-length tables)
  3. Parsable Byte Stream (text files, XML files, CSV tables)
  4. Encoded Byte Stream (PDF files, JPEG images) NOTE: Not used for science observations
The vast majority of observational data can be described as either N-dimensional arrays or tables. The type of data structure associated with your bundle is dependent upon the nature of the observational data. This decision is to be made with your node representative. To see a list of which subclasses exist for the Product_Observational class visit the Data Provider's Handbook, 6.2.2.1

When a data template is added to a bundle (whether it be 2D Array, Table Character Delimited, etc.), ELSA adds the product to the list in Product_Collection and creates a Product_Observational. The Product_Observational contains:

  • a reference list area for articles and other products relevant to understanding the product
  • file area for identifying the file(s) containing the digital object(s) and includes classes that describe each digital object in the given file (e.g., the description and parameters of each header, table, and image).

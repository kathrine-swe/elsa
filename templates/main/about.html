ELSA

Introduction.

The Educational Labeling System for Atmospheres (ELSA) is designed to help data providers create PDS4 compliant labels. ELSA's goal is to provide an online environment where data providers can learn about PDS4, build PDS4 bundles, submit completed bundles to Atmos, and manage the reviews/lien resolutions all in one space.

PDS4 Standard

Introduction to PDS4.

All data to be archived at PDS4 must have an associated set of metadata according to PDS4 Standard. The most current version of PDS4 is {{ most_current_version }}. PDS4 is the most current archiving standard for your labels and its predecessor was PDS3. The major difference between the two is how each standard dictates a label to be written. PDS4 describes metadata using eXtensible Markup Language (XML). XML is a widely available language for metadata management.

ELSA

We like the Top-Down.

At Atmospheres, we chose the Top-Down approach because it gives data providers the big picture of what a Bundle is at the very beginning of their PDS4 journey. The Top-Down approach allows Logical Identifiers (LIDs) to be created in the natural way (Data Provider's Handbook 1.7.0.0, Section 5.2). The PDS4 standard's immense power to archive and organize large data sets revolve around LIDs.

Upon registration with ELSA, the user is asked which agency they work for--NASA, ESA, etc. This creates the agency identifier for each LID throughout the Bundle. When a user begins creating a Bundle, they are asked for the Bundle's name and which Collections their Bundle has. ELSA prevents Bundles of the same name to be created to ensure that Bundles created within ELSA are unique. This provides for an area of improvement within ELSA in which one day we may expand to ensure that the Bundle in creation is unique across all registered products. Until then, data providers should consult their Node to ensure the name is allowed. Once the Bundle name is unique, then regardless of all else that follows, the entire LID is unique.

ELSA for Developers

The PDS Atmospheres Node hosts ELSA's Build application on GitHub. The main purpose is to establish a working history for the students who develop ELSA at NMSU. Feel free to report bugs, request features, and do all of the other usual things you can do with GitHub to help us better improve ELSA.
